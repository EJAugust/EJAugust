This is a simple demonstration of the way that an object will inherit its prototype value until something causes it to shadow that value.

The state1 property is created via <some object>.state1 = 0
state2 is created via Object.defineProperty(<some object>, { state2: { value: 0 } })

a and b inherit from prototype.

Before increment:

a shadows state1 and state2.
b does not.

prototype state1: 0state2: 0
astate1: 0state2: 0
bstate1: 0state2: 0

prototype.state1++

Notice how b changes but a stays the same.

prototype state1: 1state2: 0
astate1: 0state2: 0
bstate1: 1state2: 0

a.state1++

a still shadows prototype.

prototype state1: 1state2: 0
astate1: 1state2: 0
bstate1: 1state2: 0

b.state1++

Here, the increment operation reads from prototype.state1 and then writes to b.state1. Now, b shadows prototype.

prototype state1: 1state2: 0
astate1: 1state2: 0
bstate1: 2state2: 0

prototype.state1++

Demonstrating that a and b are both isolated from prototype.

prototype state1: 2state2: 0
astate1: 1state2: 0
bstate1: 2state2: 0

prototype.state1++

prototype state1: 3state2: 0
astate1: 1state2: 0
bstate1: 2state2: 0

prototype.state2++

state2 behaves the same way as state1.

prototype state1: 3state2: 1
astate1: 1state2: 0
bstate1: 2state2: 1

a.state2++

prototype state1: 3state2: 1
astate1: 1state2: 1
bstate1: 2state2: 1

b.state2++

prototype state1: 3state2: 1
astate1: 1state2: 1
bstate1: 2state2: 2

prototype.state2++

prototype state1: 3state2: 2
astate1: 1state2: 1
bstate1: 2state2: 2

prototype.state2++

prototype state1: 3state2: 3
astate1: 1state2: 1
bstate1: 2state2: 2
