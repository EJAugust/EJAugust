This demonstrates a convenient feature in JavaScript whereby a string is automatically casted to a number, thanks to the increment and decrement operations.

Initial State

Here, we have six variables which will help demonstrate the effect for us.

let a = "2"
let b = "2"
let c = "2"
let d = "2.14159"
let e = "Euler's number is not 2.14159"
let f = "2.14159 is not Euler's number"

Naturally, the typeof all four is "string".

Adding a Number to an Obvious Integer String

Here, we'll add a number directly to a using a += 1.

a === "21"
typeof a === "string"

This did the opposite of what we wanted. We wanted to convert the string to a number and increase it by one, but this converted the one to a string and concatenated it to a.

Using Increment and Decrement on Integer Strings

Next, we'll change b and c by incrementing one and decrementing the other (b++ and c--).

b === 3
typeof b === "number"
c === 1
typeof c === "number"

This is the result we were after.

Now, b and c have both changed from their original value of "2".

One went up, and one went down.

From this point on, the type of both b and c is now number.

What Happens to Float strings?

Next, we'll try to do the same thing to d, a string that looks like it could be a floating point value, using d++.

d === 3.14159
typeof d === "number"

It worked! JavaScript automatically parsed the float value for us and then increased it.

What Happens to Non-Numeric Strings?

Finally, we'll try the same trick on e and f to see what happens.

e === NaN
typeof e === "number"
f === NaN
typeof f === "number"

As you can see, both cases created the infamous NaN.
